# 移动端响应式设计文档

## 概述

本系统已实现移动端响应式适配，用户可以在手机、平板等移动设备上正常访问和使用所有功能。

## 主要特性

### 1. 汉堡菜单
- **桌面端**：左侧固定侧边栏（200px宽）
- **移动端**：顶部汉堡菜单按钮，点击后显示抽屉式导航菜单
- 移动端菜单自动在选择菜单项后关闭

### 2. 断点设置
- **移动端**：< 768px
- **平板**：768px - 991px
- **桌面端**：≥ 992px

### 3. 响应式表格
- 表格支持水平滚动，确保在小屏幕上不会出现布局问题
- 某些列在移动端自动隐藏（通过CSS类控制）
- 表头固定，操作列固定在右侧

### 4. 响应式表单
- 搜索表单在移动端自动简化为关键词搜索
- 其他筛选条件自动隐藏，用户可通过桌面端使用完整功能
- 对话框宽度自适应，百分比布局

### 5. 按钮优化
- 移动端按钮增大可点击区域
- 次要信息在移动端隐藏，只显示关键操作按钮
- 图标按钮在小屏幕上更易点击

## 响应式类名

### 全局响应式类

在HTML中可以使用以下类名控制元素在不同屏幕尺寸下的显示：

```html
<!-- 在移动端隐藏，桌面端显示 -->
<div class="hide-on-mobile">内容</div>

<!-- 只在移动端显示 -->
<div class="show-on-mobile">内容</div>

<!-- 只在平板及以上显示 -->
<div class="hide-on-tablet">内容</div>
```

### 响应式容器

```html
<div class="responsive-container">
  内容会自动根据屏幕宽度调整
</div>
```

### 响应式表格包装器

```html
<div class="table-responsive-wrapper">
  <el-table>...</el-table>
</div>
```

### 响应式表单

```html
<el-form class="responsive-form">
  ...
</el-form>
```

### Element Plus 表格列响应式

通过 `class-name` 属性设置响应式类：

```vue
<el-table-column 
  label="手机号" 
  class-name="hide-on-mobile"
/>

<el-table-column 
  label="房间号" 
  class-name="hide-on-tablet"
/>
```

## 最佳实践

### 1. 表格列优先级
建议按以下顺序显示列：
1. **必须显示**：ID、用户名、状态、操作按钮
2. **重要信息**：姓名、手机号、套餐名称
3. **次要信息**：详细时间、描述等

### 2. 按钮布局
- 主要操作（新增、搜索）始终显示
- 次要操作（导入、导出）可在移动端隐藏
- 使用下拉菜单收纳更多操作

### 3. 对话框
- 使用 `width="90%"` 或 `width="95%"` 替代固定像素宽度
- 表单项在移动端使用全宽布局

## 测试建议

1. **手机竖屏**：测试主要内容是否正常显示
2. **手机横屏**：测试布局是否自动调整
3. **平板设备**：测试是否需要显示更多列
4. **触控操作**：确保所有按钮和交互元素可正常点击

## 注意事项

1. 某些复杂表格（如网络用户详情弹窗中的费用明细）建议在移动端简化显示
2. 导出/导入功能建议在桌面端使用
3. 大量数据操作（如批量删除）建议在桌面端执行
